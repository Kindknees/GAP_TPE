# 桃機資料說明：
這邊我們有拿到桃園機場2025年6月23日的航班資料

## 桃機的作業流程
前一天晚上約10點會出來隔天的登機門指派，就如資料中的 「final bay」。到了隔天實際上陣，航空公司都會有各種要求希望可以進行調整，因此一天調整幾十次都有可能，調整後的結果就如資料中的「第81」版。

## 資料照片
第1~10張照片：final bay的資料，也就是前一天(6/22)排出來的預指派
第11~20張照片：第81版的資料，也就是經過一整天的調整後最終實際指派結果
以1.png的第一個航班為例：
- CEB310/311 是這個航班的班號
- 0100 - 0225: 會1:00 ~ 2:25會佔用登機門
再以11.png的第一個航班為例：
- JJA6154 是這個航班的班號
- 黑色字體0250D：預指派的時間，預計是2:50 depart(D 是 depart、A是 arrive)，這邊就只有給depart的時間
- 紅色字體0242D：實際的時間，表示這班航班實際上在2:42 depart的

## csv檔案
process_data.py: 將資料進行處理的python檔案，處理後的如同processed_0623_ver81.csv
標題解釋：
- flight_index：我們自己給每個航班的一個index
- flight_number：原始的班號
- original_gate_name：桃機原本給的登機門編號
- original_gate_index：我們自己給每個登機門一個編號
- start_time_str：桃機資料中寫的，航班預計佔用登機門的開始時間
- end_time_str：桃機資料中寫的，航班預計佔用登機門的結束時間
- updated_start_str：桃機資料中寫的，航班實際佔用登機門的開始時間
- updated_end_str：桃機資料中寫的，航班實際佔用登機門的結束時間
- start_time_min：將預計開始佔用登機門時間轉換成一天中的第幾分鐘
- end_time_min：將預計結束佔用登機門時間轉換成一天中的第幾分鐘
- updated_start_min：將實際開始佔用登機門時間轉換成一天中的第幾分鐘
- updated_end_min：將實際結束佔用登機門時間轉換成一天中的第幾分鐘
- service_time_min：將預計開始時間減掉預計結束時間算出來的
